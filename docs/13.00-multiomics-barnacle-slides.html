Sparse Tensor Decomposition for Multi-Species Gene Expression Analysis

Benefits vs. Pairwise Comparisons, PCA, and WGCNA

Sam White

2026-02-25

The Problem: Three Simultaneous Axes of Variation

Our coral data has structure that classical methods cannot natively capture:

Axis 1: Genes

Ortholog groups shared across species

Axis 2: Samples

Three species × biological replicates

Axis 3: Time

TP1 → TP2 → TP3 → TP4

Every classical method flattens or ignores at least one of these axes.

What Is a Tensor?

A tensor is a multidimensional array — a generalization of a matrix.

  • Scalar = 0th-order tensor
  • Vector = 1st-order tensor
  • Matrix = 2nd-order tensor
  • 3D array = 3rd-order tensor ← our data
Mode This analysis
Axis 1 Ortholog groups
Axis 2 Species-samples
Axis 3 Timepoints (TP1–4)

Tensors naturally represent datasets where the measurement (expression) depends on multilinear combinations of multiple variables simultaneously.

What Is Tensor Decomposition?

Tensor decomposition breaks a complex dataset into a set of simpler, distinct components — each representing one independent biological program.

Each component contains three linked factor vectors:

  • Gene factor — which orthologs define this program
  • Sample factor — which species/samples express it
  • Time factor — the temporal trajectory of the program

Think of it like audio source separation: one jumbled recording → individual speakers’ voices isolated into distinct channels.

The “Sparse” Part

Barnacle applies a sparsity constraint that pushes most gene weights toward zero, keeping only the strongest signals.

Without sparsity

  • Every gene has a non-zero weight
  • Background noise mixed with signal
  • Hard to define meaningful gene sets

With sparsity ✓

  • Most gene weights driven to zero
  • Non-zero genes = true signal
  • Component gene sets emerge naturally

Sparsity = interpretability. No arbitrary threshold needed to extract a gene list from a component.

What Is “Rank”?

Rank is the number of components (factors) the model extracts from the tensor.

  • Each component = one distinct biological program
  • Rank 1: the entire dataset described by a single pattern
  • Rank 5: five independent co-expression programs
  • Higher rank → more nuanced signal, but more complexity to estimate

Think of rank as the number of “channels” you are asking the model to separate the data into. The goal is to find the smallest number of channels that still captures all the real biology.

How Does Barnacle Find the Components?

Core idea: Search for combinations of genes, species, and timepoints that consistently rise and fall together — across all three axes at once.

The fitting process — iterative refinement:

Think of it like focusing a blurry photograph. The model:

  1. Starts with a rough random guess for each component
  2. Asks: “Given the current time and species patterns, which genes best match the data?” → refines the gene list
  3. Asks: “Given the updated genes and time patterns, which species/samples best match?” → refines the sample weights
  4. Asks: “Given the updated genes and samples, which time trajectory best fits?” → refines the time curve
  5. Cycles through steps 2–4 until the components stop changing — the picture is “in focus”

Two additional safeguards:

  • This is repeated many times from different random starting points — the best result across all starts is kept, avoiding accidental local solutions
  • The sparsity constraint is enforced at step 2 each cycle: gene weights that aren’t strongly supported get pushed to zero, so only the clearest signals survive

Stage 1: Selecting the Best Rank

The question: How many distinct biological programs are actually present in the data — not too few, not too many?

Analogy: Imagine grouping thousands of news articles by topic. Grouping into 3 topics (politics, sports, other) loses real distinctions. Grouping into 500 creates artificial sub-categories that won’t appear in next week’s news. The right number captures real, recurring themes.

How it works — “study half, test half”:

  1. Randomly split the data into a training half and a held-back test half
  2. Fit the model on the training half
  3. Measure how accurately it reconstructs the held-back data — lower error = the model captured real signal, not memorized noise
  4. Repeat across many random splits and average the error; do this for every candidate rank

Parsimony rule: When a simpler model is statistically just as accurate as a more complex one, always prefer the simpler one — it generalizes better and is less likely to be fitting noise.

This is objective, unlike the PCA “scree plot elbow” or WGCNA soft-thresholding power selection, both of which rely on subjective visual inspection.

Stage 2: Selecting the Best Sparsity Level

Key distinction from rank: Rank asked how many co-expression programs exist. Sparsity asks which genes truly belong to each one.

Every gene gets a score for every component. Without sparsity, even weakly-associated genes keep a non-zero weight, blurring the boundaries between programs. Sparsity sets a commitment threshold — genes that don’t meet it are dropped to zero; only genes that strongly and consistently define a component remain.

Analogy: Think of assembling project teams within a company. Rank decided there are 35 teams. Now sparsity determines who actually counts as a team member.

  • A loose threshold includes anyone who ever attended a meeting — team lists are long, overlapping, and hard to characterize
  • A strict threshold keeps only core contributors — smaller, cleaner, more meaningful teams
  • Too strict → you’ve excluded people who genuinely do the work

How it works — “do both halves name the same core members?”:

The model tries many different commitment thresholds. For each one:

  1. Split the data into two independent halves
  2. Fit the model separately on each half
  3. Ask: do both halves identify the same core genes for each component?
    • If yes → the threshold is keeping real, reproducible members
    • If no → the threshold is too strict; it’s dropping genes that genuinely define the pattern
  4. Repeat across many random splits and average the agreement

Selecting the ideal setting: Pick the strictest threshold where component gene sets still reliably match across splits — the smallest, cleanest gene sets that remain reproducible.

After the final model is fit, each of the 35 components also gets its own individual agreement score — an extra quality-control check unavailable in WGCNA or PCA.

vs. Pairwise Comparisons (DESeq2 / edgeR)

Aspect Pairwise Sparse Tensor
Data structure 2D: genes × samples 3D: genes × samples × time
Contrasts One predefined contrast at a time All axes simultaneously
Cross-species Separate tests; post-hoc overlap All species in one joint model
Temporal pattern Interaction terms or repeated tests Time is a native dimension
Output Binary gene lists Continuous loadings across all axes

Pairwise approaches require multiple tests + arbitrary overlap thresholding and cannot capture temporal trajectories as a unit.

vs. PCA

PCA is a matrix decomposition — to use it you must first flatten the tensor:

Problems with flattening

  • Collapsing time into samples conflates temporal variation with inter-individual variation
  • Running PCA per species gives no shared latent space
  • Dense loadings require post-hoc thresholding
  • Orthogonality constraint may not match biology

Tensor advantage

  • Native 3D structure preserved
  • Three linked factor matrices remain jointly meaningful
  • Sparsity constraint replaces arbitrary cutoffs
  • No orthogonality forced

vs. WGCNA

WGCNA also discovers co-expression modules without predefined contrasts — but operates only on 2D matrices.

Aspect WGCNA Sparse Tensor (Barnacle)
Data structure 2D: gene × sample 3D: gene × sample × time
Multi-species Per-species; post-hoc comparison All species in one decomposition
Temporal data Time encoded as a trait Time is a first-class axis
Module definition Topological overlap + clustering Sparsity-constrained optimization
“Dark matter” genes Often filtered out Included; placed by co-variation
Supervised element Module–trait correlations required Fully unsupervised

The WGCNA Cross-Species Problem

Important

WGCNA requires a separate run per species, then manual post-hoc comparison of module gene lists.

Any conserved pattern across species and timepoints is only findable after the fact.

With sparse tensor decomposition, cross-species conservation is directly encoded:

If a factor has high weights for orthologs in the gene axis and high weights for all three species in the sample axis, that factor directly represents a conserved cross-species expression program.

Key Advantages (1 of 2)

  1. Native multi-way structure — genes, samples, and time are first-class dimensions; no flattening required

  2. Single joint model — all three coral species analyzed simultaneously; conservation is a direct output, not post-hoc

  3. Temporal trajectory as a factor — full TP1→TP4 dynamics captured as a factor vector, not just a trait correlation

  4. Sparsity = interpretability — sparsity constraint drives most gene weights to zero; gene sets emerge without threshold tuning

Key Advantages (2 of 2)

  1. Fully unsupervised — no phenotype labels or predefined contrasts; uncharacterized “dark matter” genes included

  2. Principled parameter selection — rank and sparsity level jointly optimized via cross-validated bootstrap grid search (1SE rule)

  3. Identifiability metric — Factor Match Score (FMS) formally quantifies component reproducibility across random initializations

Principled Parameter Selection

Two-stage bootstrap grid search:

Stage 1 — Rank selection

  1. Fit models across candidate ranks
  2. Measure prediction error across bootstrap splits
  3. Select smallest rank within statistical equivalence of best
  4. Parsimony principle

Stage 2 — Sparsity selection

  1. Fit models across candidate sparsity levels
  2. Measure component stability (Factor Match Score) across splits
  3. Select maximum sparsity that remains stable
  4. Maximum interpretability while stable

This is a key advantage over WGCNA (soft-thresholding power choice) and PCA (scree plot elbow) — both of which rely on subjective visual inspection.

What a Factor Looks Like

Each component simultaneously describes:

Component k:
  Gene factor:   [0, 0, 0.83, 0, 0, 0.61, 0, 0, 0.44, ...]  ← sparse gene set
  Sample factor: [0.71, 0.68, 0, 0.73, 0.69, 0, 0.65, ...]   ← which species/samples
  Time factor:   [0.12, 0.87, 0.84, 0.15]                    ← TP1-low, TP2-3-high

Reading this factor: a sparse set of orthologs is up-regulated in two of the three coral species with a peak at TP2–TP3 — a conserved, time-resolved co-expression program discovered without any labels.

Application: E5 Timeseries Coral Data

Tensor dimensions:

  • Axis 1: Ortholog groups (A. pulchra × P. evermanni × P. tuahiniensis)
  • Axis 2: Combined species-samples (all replicates)
  • Axis 3: TP1, TP2, TP3, TP4

Normalization: sctransform (regularized negative binomial)

Tool: Barnacle (Blaskowski et al. 2024)

Scientific question:

Which gene expression programs are conserved across evolutionarily divergent coral species at the same developmental timepoints?

This question is not directly answerable by pairwise tests, PCA, or per-species WGCNA.
